pre-commit es un framework que ejecuta validaciones automaticas antes de permitir un commit en Git.
Esto garantiza que todo el codigo enviado al repositorio cumpla con los estandares del proyecto, evitando errores comunes y manteniendo un estilo consistente.

en fusion-worflow se usa para:

Limpiar espacios innecesarios
asegurar fin de archivo correcto
validar archivos YAML
evitar archivos gigantes por accidente
formatear codigo Python con Black
